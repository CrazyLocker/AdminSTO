# Инструкция по использованию приложения «Администратор СТО»

## Общее описание

**Администратор СТО** — десктопное приложение для управления автосервисом. Предназначено для учёта клиентов, заказов (наряд-заказов), услуг, запасных частей, склада и записи клиентов на обслуживание.

### Технологический стек

| Компонент | Технология |
|-----------|------------|
| Язык | Java 21 |
| UI-фреймворк | JavaFX 21.0.6 |
| База данных | SQLite (файловая) 3.44.1.0 |
| Пул соединений | HikariCP 5.0.1 |
| Сборка | Gradle 8.7 |
| Тестирование | JUnit 5.10.1 + Mockito 5.8.0 + AssertJ 3.25.1 |
| Логирование | SLF4J + Logback 1.4.14 |
| Отчёты (PDF) | iText 7.2.5 |
| Сериализация | Gson 2.10.1 |

---

## Запуск приложения

### Из IDE (IntelliJ IDEA, OpenIDE и др.)

1. Откройте проект в корневой директории `AdminSTO`.
2. Убедитесь, что JDK 17 настроен как SDK проекта.
3. Найдите класс `com.autoservice.App` (файл `App.java`).
4. Запустите метод `main(String[] args)`.

Приложение автоматически создаст базу данных SQLite при первом запуске.

### Из командной строки (Maven)

```bash
# Сборка проекта
mvn clean package

# Запуск
mvn javafx:run
```

### Сборка исполняемого JAR

```bash
./gradlew fatJar
```

Результат: `build/libs/autoservice-admin.jar` (fat JAR со всеми зависимостями).

## Вкладки приложения

Приложение состоит из 8 вкладок, расположенных в верхней части окна:

---

### 1. 📊 Дашборд (главная страница)

Сводная информация о работе автосервиса.

**Что отображается:**

| Карточка | Описание |
|----------|----------|
| **Заказов** | Общее количество заказов |
| **Клиентов** | Общее количество клиентов |
| **Низкие остатки запчастей** | Количество запчастей с остатком ниже минимального |
| **Выручка** | Сумма всех закрытых заказов |
| **В работе** | Количество заказов со статусом «В работе» |
| **Выполнено** | Количество закрытых заказов |
| **Записей** | Количество активных записей |

**Быстрые действия:**

| Кнопка | Действие |
|--------|----------|
| **Новый заказ** | Открыть диалог создания нового заказа |
| **Новый клиент** | Открыть диалог создания нового клиента |
| **Запись** | Переключиться на вкладку «Запись» |
| **Отчёт** | Открыть генератор отчётов (PDF) |

**Таблица «Записи на текущую неделю»:**
- Показывает все активные записи на текущую неделю (с понедельника по воскресенье).
- **Двойной клик** по строке — просмотр связанного заказа.
- Цветовая индикация: 🟦 Синий — новый, 🟧 Оранжевый — в работе, 🟩 Зелёный — закрыт.

### 2. 👥 Клиенты

Управление базой клиентов.

**Колонки таблицы:**

| Колонка | Описание |
|---------|----------|
| Фамилия | Фамилия клиента |
| Имя | Имя клиента |
| Телефон | Контактный телефон |
| Автомобиль | Модель и государственный номер |
| Последний ремонт | Дата последнего обращения |

**Действия:**

| Кнопка / Горячая клавиша | Действие |
|--------------------------|----------|
| **Новый клиент** / `Ctrl+N` | Создать нового клиента |
| **Изменить** / `Ctrl+S` | Редактировать выбранного клиента |
| **Удалить** / `Delete` | Удалить клиента (с подтверждением) |
| **Поиск** | Фильтрация по имени, фамилии, телефону, номеру авто, модели |
| `Escape` | Очистить поиск |
| Двойной клик | Открыть диалог редактирования |

**Диалог клиента:**
- Обязательные поля: Фамилия, Имя, Телефон (формат: +7 (XXX) XXX-XX-XX).
- Необязательные: Автомобиль (модель), Гос. номер, Примечание.
- Поддерживается выпадающий список моделей: Haval, GWM, Great Wall.

### 3. 📋 Заказы (Наряд-заказы)

Управление заказами на обслуживание.

**Колонки таблицы:**

| Колонка | Описание |
|---------|----------|
| № заказа | Уникальный номер (ZAK-ДД-ММ-ГГ-XXXX) |
| Клиент | ФИО клиента |
| Автомобиль | Модель и гос. номер |
| Услуги | Список услуг в заказе |
| Запись | Дата и время записи |
| Сумма | Общая сумма заказа |
| Статус | Выпадающий список: Новый / В работе / Закрыт |

**Статусы заказов:**

| Статус | Описание |
|--------|----------|
| **Новый** | Заказ только создан |
| **В работе** | Обслуживание выполняется |
| **Закрыт** | Заказ завершён |

**Действия:**

| Кнопка / Горячая клавиша | Действие |
|--------------------------|----------|
| **Новый** / `Ctrl+N` | Создать заказ (выбор клиента + услуги + запчасти) |
| **Изменить** / `Ctrl+S` | Редактировать заказ |
| **Удалить** / `Delete` | Удалить заказ (с подтверждением) |
| **Печать** | Печать наряда-заказа в PDF |
| **Поиск** | Фильтрация по клиенту, телефону, номеру авто |
| `Escape` | Очистить поиск |
| Двойной клик | Открыть диалог редактирования |
| Клик по статусу | Изменение статуса без открытия диалога |

**Расширенные фильтры** (кнопка «Расширенный фильтр»):
- **Статус**: Все / Новый / В работе / Закрыт
- **Дата от / Дата до**: Фильтр по дате создания заказа
- **Сумма от / Сумма до**: Фильтр по сумме заказа
- **Сбросить**: Сброс всех фильтров

### 7. 📅 Запись

Календарь записей клиентов на обслуживание.

**Виды календаря:**

| Вид | Описание |
|-----|----------|
| **Неделя** | Сетка 7 дней × 13 временных слотов (08:00–20:00) |
| **Месяц** | Представление месяца с мини-записями |

**Навигация:**

| Элемент | Действие |
|---------|----------|
| **Сегодня** | Вернуться к текущей дате |
| **◀ / ▶** | Предыдущая / следующая неделя |
| **Календарь** | Выбор конкретной даты |

**Мастера:** Саныч, Малой (загружаются из настроек).

**Цветовая индикация ячеек:**

| Цвет | Значение |
|------|----------|
| ⬜ Белый | Свободно |
| 🟦 Синий | Запись создана, заказ «Новый» |
| 🟧 Оранжевый | Заказ «В работе» |
| 🟩 Зелёный | Заказ «Закрыт» |
| ⬜ Серый | Запись есть, но заказ не найден |

**Действия:**

| Действие | Результат |
|----------|----------|
| **Двойной клик по свободной ячейке** | Создание новой записи |
| **Двойной клик по занятой ячейке** | Просмотр деталей записи |
| **Drag & Drop** | Перемещение записи на другое время/день |
| **Двойной клик на записи в месяце** | Просмотр информации о записи |

**Создание записи:**
1. Выберите заказ из списка (доступны только открытые заказы).
2. Выберите мастера.
3. Укажите дату и время.
4. Система проверит занятость времени и выходной день.

## Форматы данных

### Телефон
Формат: `+7 (XXX) XXX-XX-XX`

### Номер заказа
Формат: `ZAK-ДД-ММ-ГГ-XXXX`
- `ДД` — день, `ММ` — месяц, `ГГ` — год, `XXXX` — порядковый номер

### Формат даты
Внутренний формат: `дд.мм.гггг` (например, `18.07.2026`)

### Формат времени в записи
`ЧЧ-мм` (например, `14-00`)

## Хранение данных

- **База данных**: SQLite, файл создаётся автоматически при первом запуске.
- **Кэш в памяти**: Данные загружаются при старте и сохраняются при закрытии.
- **Состояние окон**: Размеры и позиция окон сохраняются автоматически (JSON).
- **Состояние таблиц**: Ширина колонок и сортировка сохраняются между сессиями.
