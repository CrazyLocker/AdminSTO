# План улучшения UI тестов проекта AdminSTO

## 📊 Текущее состояние

### Анализ
В проекте есть 24 тестовых класса с ~290 тестами. Статус тестирования:

| Уровень | Классов | Тестов | Статус |
|---------|---------|--------|--------|
| Unit | 20+ | ~250 | ✅ Пройдены |
| UI (JavaFX) | 1 (BaseUITest) | 0 активных | ⚠️ `@Disabled` |
| Integration | 3+ | ~40 | ⚠️ 1 падает |

### Проблемы

1. **UI тесты не работают**:
   - `BaseUITest` не используется другими тестами
   - Все UI тесты классов (CreateOrderDialogTest, EditClientDialogTest и др.) тестируют только модели, а не реальный JavaFX UI
   - Тег `@Tag(TestTags.UI)` не означает реальное тестирование UI

2. **Падающий тест**:
   - `ImportServiceTest.testImportFromXmlWithEncoding()` - XML импорт не парсится (возвращает 0 вместо 1)

3. **Отсутствуют автотесты**:
   - Нет реальных JavaFX UI тестов (таблицы, диалоги, вкладки)
   - Нет E2E тестов полных сценариев
   - Нет тестов графического интерфейса

---

## 🎯 Цель

Сделать UI тесты полноценными - добавить реальное тестирование JavaFX интерфейса с использованием TestFX и headless режима.

**Тест**: `src/test/java/com/autoservice/services/ImportServiceTest.java`

### Шаг 2: Улучшить BaseUITest

**Файл**: `src/test/java/com/autoservice/BaseUITest.java`

**Текущие проблемы**:
- Robot класс эмулирует только задержки, не отправляет реальные события
- Нет реального взаимодействия с JavaFX узлами
- Метод `getNodeCenter()` использует bounds, которые могут быть неверными в headless режиме

**Улучшения**:
1. Заменить кастомный `Robot` на реальный `javafx.scene.robot.Robot`
2. Использовать `TestFX` API для более надежного взаимодействия
3. Добавить проверку, что JavaFX Platform запущен
4. Использовать `Platform.runLater()` для всех операций с UI
5. Добавить таймауты и ожидания загрузки

**Код**:
```java
// Заменить на:
import javafx.scene.robot.Robot;
// И использовать реальный Robot для кликов
```

## 🧪 Типы UI тестов

### 1. Unit-UI тесты (модульные)
- Тестирование контроллеров с mock UI
- Проверка обработчиков событий
- Проверка валидации

### 2. Integration UI тесты
- Полный цикл: UI → контроллер → модель → БД
- Проверка переключения вкладок
- Проверка сохранения состояния

### 3. E2E тесты
- Полные сценарии: "Клиент → Заказ → Запись → Закрытие"
- Тестирование всех горячих клавиш
- Тестирование импорта/экспорта

## 📝 Примечания

1. **TestFX** - библиотека для тестирования JavaFX уже добавлена в зависимости (`testImplementation 'org.testfx:testfx-core:4.0.18'`)

2. **Headless режим** - JavaFX можно запускать без GUI в режиме software rendering

3. **Изоляция тестов** - каждый тест должен очищать БД и сбрасывать кэш

4. **Таймауты** - использовать `awaitility` для асинхронных операций

5. **Визуализация** - для отладки можно отключить headless режим и видеть тесты
