# Рекомендации по перезагрузке IDE

## ✅ Сессия сохранится при:
- Перезагрузке IDE (Ctrl+Shift+R)
- Закрытии/открытии проекта
- Обновлении Maven зависимостей
- Переключении веток Git

## ⚠️ Сессия может сброситься при:
- Удалении папки `.gigacode`
- Переустановке GigaCode
- Сбросе настроек IDE
- Удалении кэша Maven

## 📁 Файлы, которые сохраняют контекст:
```
.gigacode/
├── session-summary.md          ← Резюме текущей сессии
├── plans/                      ← Планы улучшений
│   └── 1783050707005-*.plan.md
└── context/                    ← Контекст проекта
    └── project-info.md
```

## 🔧 Чтобы восстановить сессию после сброса:

1. Откройте проект `AdminSTO`
2. Скажите: "Продолжи с последней сессии"
3. Или укажите файл `.gigacode/session-summary.md`

## 📝 Полезные команды:

```
"Покажи историю изменений"
"Что мы делали в последней сессии?"
"Продолжи миграцию БД"
"Запусти тесты"
"Покажи архитектуру БД"
```

## 🎯 Следующие шаги (если нужно):

1. **Исправить JavaFX threading проблемы** (Controller тесты)
2. **Исправить ImportServiceTest** (null file handling)
3. **Исправить StatisticsServiceTest** (empty revenue)
4. **Добавить покрытие View-слоя** (если требуется)

## 📚 Документация:
- `CHANGELOG.md` - изменения версий
- `DEVELOPMENT_PLAN.md` - план разработки
- `TECHNICAL_SPECIFICATION.md` - техническое задание
- `TESTING.md` - инструкция по тестированию

---

**Важно:** Сессия GigaCode хранится в облаке и не зависит от состояния вашей IDE.
Даже если вы закроете все окна, при следующем запуске GigaCode вы сможете продолжить работу.
