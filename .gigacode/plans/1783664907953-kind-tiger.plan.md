### ValidationErrorIndicator.java
**Расположение:** `src/main/java/com/autoservice/utils/ValidationErrorIndicator.java`

**Методы:**
- `showError(Field field, String message)` - показать ошибку на поле (красная рамка + tooltip)
- `clearError(Field field)` - очистить ошибку на поле
- `clearAllErrors(Node root)` - очистить все ошибки в корневом элементе (GridPane, VBox, HBox)

**Использование:**
```java
// Показать ошибку
ValidationErrorIndicator.showError(field, "Сообщение об ошибке");

// Очистить ошибку
ValidationErrorIndicator.clearError(field);

// Очистить все ошибки перед новой валидацией
ValidationErrorIndicator.clearAllErrors(root);
```

## 3. Принципы интеграции валидации

### Паттерн валидации для форм:

```java
saveBtn.setOnAction(e -> {
    // 1. Очистить все ошибки перед валидацией
    ValidationErrorIndicator.clearAllErrors(root);
    
    boolean isValid = true;
    
    // 2. Валидировать каждое поле
    if (!ValidationUtils.isValidX(field.getText())) {
        ValidationErrorIndicator.showError(field, "Сообщение об ошибке");
        isValid = false;
    }
    
    // 3. Вернуться, если есть ошибки
    if (!isValid) {
        return;
    }
    
    // 4. Сохранять данные только если валидно
    // ... сохранение данных
});
```

### Паттерн добавления tooltip:

```java
// При создании поля
TooltipHelper.setToolTip(field, "Описание формата или инструкция");

// Для ComboBox
TooltipHelper.setToolTip(comboBox, "Выберите значение");
```

## 4. Требования к валидации

1. **Всегда очищать ошибки** перед новой валидацией через `ValidationErrorIndicator.clearAllErrors(root)`
2. **Использовать ValidationUtils** для всех проверок:
   - `isNotBlank()` для обязательных текстовых полей
   - `isValidPhone()` для телефонов
   - `isValidCarNumber()` для госномеров
   - `isPositiveInteger()` / `isNonNegativeInteger()` для целых чисел
   - `isNonNegativeDouble()` для вещественных чисел
   - `isValidEnum()` для выпадающих списков
3. **Показывать ошибки** через `ValidationErrorIndicator.showError(field, message)`
4. **Добавлять tooltip** через `TooltipHelper.setToolTip(field, text)` для подсказок пользователю
5. **Возвращаться из обработчика** сразу после обнаружения первой ошибки (или собрать все ошибки и показать их)

## 7. Структура валидации

```
Form
├── ValidationErrorIndicator (визуальная индикация)
│   ├── showError() - красная рамка + tooltip
│   ├── clearError() - убрать красную рамку
│   └── clearAllErrors() - убрать все ошибки
├── ValidationUtils (логика проверок)
│   ├── isNotBlank() - не пустое
│   ├── isValidPhone() - формат телефона
│   ├── isValidCarNumber() - формат госномера
│   ├── isValidEmail() - формат email
│   ├── isValidEnum() - в списке допустимых
│   └── ...
├── TooltipHelper (подсказки)
│   ├── setToolTip() - установить подсказку
│   ├── clearTooltip() - удалить подсказку
│   └── updateTooltip() - обновить подсказку
└── Alert (сообщения об ошибках)
    └── showAlert() - всплывающее окно (используется для сложных ошибок)
```
