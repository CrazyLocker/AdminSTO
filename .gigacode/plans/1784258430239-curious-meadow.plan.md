# План исправления сортировки колонки "№ заказа"

## Проблема
Сортировка в колонке "№ заказа" работает некорректно. Номер заказа имеет формат: `ZAK-dd-MM-yy-0001`
- Пример: `ZAK-10-07-26-0001`
- Паттерн: `ZAK` + дата (dd-MM-yy) + порядковый номер (0001)

Сейчас сортировка лексикографическая (по всей строке), что приводит к неправильному порядку.

## Требование
Сортировка должна учитывать:
1. Сначала дату создания (dd-MM-yy)
2. Затем порядковый номер (0001)

## Анализ кода

### Текущая реализация
**Файл:** `src/main/java/com/autoservice/views/OrderView.java`

Сейчас используется стандартный механизм JavaFX `SortedList` с привязкой к `orderTable.comparatorProperty()`:
```java
sortedData = new SortedList<>(masterData);
sortedData.comparatorProperty().bind(orderTable.comparatorProperty());
orderTable.setItems(sortedData);
```

Колонка "№ заказа" настроена как сортируемая:
```java
TableColumn<WorkOrder, String> colId = new TableColumn<>("№ заказа");
colId.setSortable(true);
colId.setCellValueFactory(new PropertyValueFactory<>("id"));
```

### Решение
Необходимо создать пользовательский компаратор для сортировки по колонке "№ заказа", который будет:
1. Извлекать дату и порядковый номер из строки номера заказа
2. Сначала сортировать по дате
3. Затем по порядковому номеру

## Реализация

### Шаги:
1. Создать компаратор для WorkOrder, который правильно парсит номер заказа
2. Применить этот компаратор к колонке "№ заказа"

### Изменяемые файлы:
- `src/main/java/com/autoservice/views/OrderView.java` - добавить пользовательский компаратор для колонки id

### Подход:
Для колонки "№ заказа" установить кастомный компаратор через `colId.setComparator()`, который будет:
- Парсить строку номера заказа (формат `ZAK-dd-MM-yy-0001`)
- Извлекать дату и номер
- Сравнивать сначала по дате, потом по номеру
