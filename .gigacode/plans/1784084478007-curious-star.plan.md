# План: Исправление пустого списка моделей в TO Parts и обновление документации

## Проблема

1. **Пустой список моделей**: В диалоге "Изменить Расходники ТО" список моделей автомобилей был пустым, что затрудняло выбор модели.

2. **Отсутствие редактируемости**: Невозможно было ввести кастомную модель автомобиля.

3. **Неправильная настройка бэкапов**: Параметры времени и количества хранимых бэкапов не сохранялись корректно.

4. **Неполные бэкапы**: Резервные копии не включали состояния таблиц (config/table-state/*.json).

## Решение

### 1. Исправление пустого списка моделей (TO Parts)

**Класс**: `SettingsView.java`

**Изменения**:
- Добавлен константный массив `GWM_MODELS` с 13 предопределёнными моделями:
  - Haval Jolion, F7, F7x, Dargo, Big Dog, H6, H9, Poer, Tank 300, Tank 500, Wingle 7, Cannon, Great Wall Poer
- Сделан ComboBox редактируемым: `modelCombo.setEditable(true)`
- Обновлен метод `getCarModelsObservable()` для использования `GWM_MODELS`
- Обновлён метод `showEditToPartDialog()`:
  - Удалена колонка "Примечание" из таблицы
  - Удалено свойство `note` из `SparePartWithQuantity`
  - Исправлено получение значения модели: `getValue()` / `getEditor().getText()`

### 2. Исправление сохранения настроек бэкапов

**Класс**: `SettingService.java`

**Новые методы**:
```java
- isAutoBackupEnabled()        // Проверка включено ли авто-бэкап
- setAutoBackupEnabled(boolean) // Включение/выключение авто-бэкапа
- getBackupTime()              // Получение времени бэкапа (HH:mm)
- setBackupTime(String)        // Установка времени бэкапа
- getBackupRetention()         // Получение количества хранимых бэкапов
- setBackupRetention(int)      // Установка количества хранимых бэкапов
- saveBackupSettings()         // Сохранение всех настроек бэкапа
```

**Класс**: `ScheduleService.java`

**Изменения**:
- Обновлен метод `loadSettings()` для использования `SettingService` вместо `System.setProperty`

**Класс**: `SettingsController.java`

**Изменения**:
- Обновлен метод `saveBackupSettings()` для использования `SettingService`

### 3. Добавление состояний таблиц в бэкап

**Класс**: `BackupService.java`

**Изменения в методе `createBackup()`**:
- Добавлено копирование файлов `config/table-state/*.json` в архив
- Структура бэкапа теперь включает:
  ```
  backup_YYYY-MM-DD_HH-mm-ss.zip
  ├── autoservice.db
  └── table-state/
      ├── clientTable.json
      ├── orderTable.json
      ├── serviceSparePartsTable.json
      ├── servicesTable.json
      ├── settingsTable.json
      ├── sparePartsTable.json
      ├── stockTable.json
      └── toPartsTable.json
  ```

   - Открыть заказ-наряд

   - Включить/выключить авто-бэкап
   - Установить время бэкапа (например, "02:00")

   - Перезапустить приложение
   - Проверить, что настройки сохранились

4. **Проверка бэкапа**:
   - Создать ручной бэкап
   - Проверить содержимое ZIP-архива
