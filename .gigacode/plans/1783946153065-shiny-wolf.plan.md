# План модернизации OrderView — Добавление сохранения настроек таблицы

## Анализ текущего состояния

### Выявленные проблемы

1. **Отсутствует фильтрация с сортировкой** — OrderView использует `SortedList<WorkOrder>` напрямую с `masterData`, без `FilteredList`
2. **Отсутствует загрузка сохранённого состояния** — нет вызова `TableStateManager.loadTableState()`
3. **Отсутствует сохранение состояния при закрытии** — в `App.java` уже есть вызов для `OrderView.getTable()`, но OrderView не готов к этому
4. **Метод `refreshOrderList()` некорректен** — он просто вызывает `applyFilters()`, который обновляет `masterData`, но не пересоздаёт `SortedList`

### Сравнение с ClientView (эталонная реализация)

| Элемент | ClientView (правильно) | OrderView (сейчас) |
|---------|------------------------|-------------------|
| Поля | `FilteredList`, `SortedList`, `masterData` | Только `SortedList`, `masterData` |
| Набор данных | `FilteredList → SortedList → TableView` | `SortedList → TableView` (без фильтрации) |
| Загрузка состояния | `Platform.runLater → TableStateManager.loadTableState()` | Отсутствует |
| Refresh | Пересоздаёт `SortedList` и заново делает `bind()` | Просто вызывает `applyFilters()` |

## План изменений

### Шаг 1. Обновить поля класса OrderView

**Файл:** `src/main/java/com/autoservice/views/OrderView.java`

**Добавить:**
```java
private static FilteredLis<WorkOrder> filteredOrders;
private static SortedList<WorkOrder> sortedOrders;  // rename existing sortedData
private static ObservableList<WorkOrder> masterData = FXCollections.observableArrayList();
```

**Удалить:**
- `private static SortedList<WorkOrder> sortedData;` (переименовать в `sortedOrders`)

---

### Шаг 2. Изменить инициализацию в методе `create()`

**Где:** Внутри `create()`, после `setupTableColumns()`

**Текущий код:**
```java
sortedData = new SortedList<>(masterData);
sortedData.comparatorProperty().bind(orderTable.comparatorProperty());
orderTable.setItems(sortedData);

refreshOrderList();
```

**Новый код:**
```java
// Создаём FilteredList для фильтрации данных
filteredOrders = new FilteredList<>(masterData, p -> true);

// Создаём SortedList для поддержки сортировки
sortedOrders = new SortedList<>(filteredOrders);
sortedOrders.comparatorProperty().bind(orderTable.comparatorProperty());
orderTable.setItems(sortedOrders);

refreshOrderList();

// ЗАГРУЗКА СОСТОЯНИЯ — отложена до после отрисовки
Platform.runLater(() -> {
    if (orderTable != null) {
        TableStateManager.loadTableState(orderTable, "orderTable");
    }
});
```

---

### Шаг 3. Обновить метод `refreshOrderList()`

**Текущий код:**
```java
public static void refreshOrderList() {
    if (orderTable != null) {
        applyFilters();
    }
}
```

**Новый код:**
```java
public static void refreshOrderList() {
    if (orderTable != null) {
        // Пересоздаём FilteredList и SortedList при обновлении данных
        masterData = FXCollections.observableArrayList(DataStore.getOrders());
        filteredOrders = new FilteredList<>(masterData, p -> true);
        sortedOrders = new SortedList<>(filteredOrders);
        sortedOrders.comparatorProperty().bind(orderTable.comparatorProperty());
        orderTable.setItems(sortedOrders);
        
        // Применяем текущие фильтры
        applyFilters();
    }
}
```

---

### Шаг 4. Обновить метод `applyFilters()`

**Где:** В конце метода, перед установкой данных в `masterData`

**Текущий код (последние строки):**
```java
masterData.clear();
masterData.addAll(filtered);
orderTable.refresh();
```

**Новый код:**
```java
// Для FilteredList нужно использовать setPredicate(), а не clear/add
// Но так как мы пересоздаём masterData в refreshOrderList(),
// здесь просто обновляем FilteredList
if (filteredOrders != null) {
    if (filtered.isEmpty()) {
        filteredOrders.setPredicate(p -> false); // нет результатов
    } else {
        final List<WorkOrder> finalFiltered = filtered;
        filteredOrders.setPredicate(order -> finalFiltered.contains(order));
    }
}
```

**Альтернативный подход (более простой):**
Так как `masterData` пересоздаётся в `refreshOrderList()`, можно просто обновлять `masterData` и вызывать `applyFilters()` для фильтрации через `Predicate`:

```java
// Вместо masterData.clear()/addAll(), обновляем FilteredList напрямую
masterData.clear();
masterData.addAll(filtered);
filteredOrders.setPredicate(p -> true); // сброс фильтра после refresh
```

## Критические моменты

1. **FilteredList обязательна** — без неё фильтрация не будет работать совместно со сортировкой
2. **Platform.runLater для loadTableState** — без этого состояние сбросится при первом layout pass
3. **Пересоздание SortedList в refresh** — иначе новые данные не подхватятся
4. **UNCONSTRAINED_RESIZE_POLICY** — уже установлен, не менять
5. **ID колонок** — уже заданы (`colOrderId`, `colClient`, `colCar`, `colServices`, `colTotal`, `colStatus`)
