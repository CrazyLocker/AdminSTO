# План: Сохранение и восстановление настроек таблиц (JavaFX TableView)

**Дата создания:** 12.07.2026  
**Задача:** Реализовать сохранение и восстановление настроек таблиц через JTablePersistenceManager для автономного приложения (запуск с флешки).

---

## 1. Анализ фреймворка JTablePersistenceManager

### Что такое JTablePersistenceManager?
- Библиотека для Java Swing (javax.swing.JTable)
- Сохраняет: ширину колонок, порядок колонок, видимость колонок
- Формат сохранения: XML

### Проблема совместимости
**⚠️ КРИТИЧЕСКАЯ ПРОБЛЕМА:** JTablePersistenceManager работает только с **Swing JTable**, а в проекте используется **JavaFX TableView**.

| Технология | Пакет | Совместимость |
|------------|-------|---------------|
| Swing JTable | javax.swing.JTable | ✅ JTablePersistenceManager |
| JavaFX TableView | javafx.scene.control.TableView | ❌ НЕ совместим |

### Вывод
**JTablePersistenceManager нельзя использовать напрямую.** Нужно либо:
1. Использовать аналог для JavaFX (отсутствует в стандартной библиотеке)
2. Реализовать собственное решение для JavaFX TableView
3. Использовать универсальный подход (JSON + Gson)

### 4.3 Сохранение при изменении
Для каждого TableView добавить listeners:

```java
// При изменении ширины колонки
table.getColumns().forEach(col -> {
    col.prefWidthProperty().addListener((obs, oldVal, newVal) -> {
        TableSettingsManager.saveTableSettings(table, tableName);
    });
});

// При изменении порядка колонок
table.getColumns().addListener((ListChangeListener.Change<? extends TableColumn<?, ?>> c) -> {
    TableSettingsManager.saveTableSettings(table, tableName);
});

// При изменении видимости колонки
table.getColumns().forEach(col -> {
    col.visibleProperty().addListener((obs, oldVal, newVal) -> {
        TableSettingsManager.saveTableSettings(table, tableName);
    });
});
```

## 7. Решение вопросов

### Вопрос 1: Как сохранять порядок колонок при перетаскивании?
**Решение:** Использовать `column.getDisplayIndex()` для получения текущего порядка. Добавить listener на `table.getColumns()` для отслеживания изменений порядка.

### Вопрос 2: Когда сохранять настройки?
**Решение:** 
- При каждом изменении (ширина, порядок, видимость) - сразу сохранять
- Дополнительно: при закрытии приложения

### Вопрос 3: Что делать, если настройки не найдены?
**Решение:** Использовать значения по умолчанию (ширины из `setPrefWidth()` в методах `createClientTable()` и т.д.)

### Вопрос 4: Поддержка разных TableView?
**Решение:** Использовать `tableName` как идентификатор. Можно сохранять настройки для каждой таблицы отдельно в одном JSON-файле:

```json
{
  "ClientView": { "columns": [...] },
  "OrderView": { "columns": [...] },
  "NewView": { "columns": [...] }
}
```

### 9.3 TableSettingsManager.java (основные методы)
```java
package com.autoservice.utils;

import com.autoservice.model.ColumnState;
import com.autoservice.model.TableState;
import javafx.collections.ObservableList;
import javafx.scene.control.TableColumn;
import javafx.scene.control.TableView;

import java.io.*;
import java.nio.file.*;
import java.util.*;

public class TableSettingsManager {
    private static final String CONFIG_DIR = "config";
    private static final String SETTINGS_FILE = "table_settings.json";
    private static final Path CONFIG_PATH = Paths.get(CONFIG_DIR, SETTINGS_FILE);
    
    private static Gson gson = new GsonBuilder().setPrettyPrinting().create();
    
    public static void saveTableSettings(TableView<?> tableView, String tableName) {
        TableState state = new TableState();
        state.setTableName(tableName);
        state.setColumns(extractColumnStates(tableView));
        
        try (Writer writer = Files.newBufferedWriter(CONFIG_PATH)) {
            gson.toJson(state, writer);
        } catch (IOException e) {
            LoggerManager.getLogger().error("Ошибка сохранения настроек таблицы", e);
        }
    }
    
    public static void loadTableSettings(TableView<?> tableView, String tableName) {
        if (!Files.exists(CONFIG_PATH)) return;
        
        try (Reader reader = Files.newBufferedReader(CONFIG_PATH)) {
            TableState state = gson.fromJson(reader, TableState.class);
            
            if (state != null && tableName.equals(state.getTableName())) {
                restoreColumnStates(tableView, state.getColumns());
            }
        } catch (IOException e) {
            LoggerManager.getLogger().error("Ошибка загрузки настроек таблицы", e);
        }
    }
    
    private static List<ColumnState> extractColumnStates(TableView<?> tableView) {
        List<ColumnState> states = new ArrayList<>();
        ObservableList<TableColumn<?, ?>> columns = tableView.getColumns();
        
        for (TableColumn<?, ?> col : columns) {
            ColumnState state = new ColumnState();
            state.setName(col.getText());
            state.setWidth(col.getPrefWidth());
            state.setOrder(col.getDisplayIndex());
            state.setVisible(col.isVisible());
            states.add(state);
        }
        return states;
    }
    
    private static void restoreColumnStates(TableView<?> tableView, List<ColumnState> states) {
        // Восстановить видимость
        Map<String, ColumnState> stateMap = new HashMap<>();
        for (ColumnState s : states) {
            stateMap.put(s.getName(), s);
        }
        
        for (TableColumn<?, ?> col : tableView.getColumns()) {
            ColumnState state = stateMap.get(col.getText());
            if (state != null) {
                col.setPrefWidth(state.getWidth());
                col.setVisible(state.isVisible());
            }
        }
        
        // Восстановить порядок
        // (требует дополнительной логики)
    }
}
```
