## ⚠️ Проблема с TestFX

### Описание:
**TestFX** версий 4.0.15, 4.0.16, 4.0.18, 5.0.0 недоступен в Maven Central. Артефакты `testfx-core` и `testfx-junit5` доступны через JitPack, но `testfx-toolkit` отсутствует в публичных репозиториях.

### Решение:
Использовать **ручную инициализацию JavaFX** вместо TestFX:
- Удалить зависимость от `testfx-toolkit` (не обязательна)
- Создать `BaseUITest` без использования `AppExtension` и `@Start/@Stop`
- Использовать JavaFX API напрямую для тестирования UI

### Текущая конфигурация (pom.xml):
```xml
<!-- TestFX для UI-тестов JavaFX -->
<!-- TestFX не доступен в Maven Central - добавлен репозиторий JitPack -->
<dependency>
    <groupId>org.testfx</groupId>
    <artifactId>testfx-core</artifactId>
    <version>4.0.18</version>
    <scope>test</scope>
</dependency>
```

**Файл:** `pom.xml`

Добавить зависимости (через JitPack repository):

### Шаг 2: Создать BaseUITest.java ✅ (ВЫПОЛНЕНО)

**Файл:** `src/test/java/com/autoservice/BaseUITest.java`

Основные возможности:
- Инициализация JavaFX в headless режиме
- Старт приложения в тестовом режиме
- Доступ к Stage и UI-элементам
- Очистка БД перед каждым тестом
- Использует `Platform.runLater()` для асинхронных операций

**Примечание:** Вместо TestFX `AppExtension` используется ручная инициализация JavaFX.

## 🔍 Ключевые вызовы и решения

### Вызов 1: JavaFX HEADLESS режим
**Проблема:** TestFX требует GUI-сессию

**Решение:**
- Использовать `System.setProperty("testfx.robot", "glass")`
- Или запускать в CI с виртуальным дисплеем (Xvfb)

### Вызов 2: Асинхронность JavaFX
**Проблема:** UI обновляется асинхронно

**Решение:**
- Использовать `Platform.runLater()` в тестах
- Применять `interactions().thenReturn()` для ожидания

### Вызов 3: База данных
**Проблема:** UI-тесты должны использовать чистую БД

**Решение:**
- Наследовать от BaseTest
- Очищать БД в `@BeforeEach`
- Использовать H2 для тестов

### Использование ручной инициализации вместо TestFX:
```java
// Вместо @Start от TestFX
void onStart(Stage stage) {
    createUI(stage);
    stage.show();
}

// Используем ручной запуск
protected Stage createStage() {
    Stage stage = new Stage();
    createUI(stage);
    stage.show();
    return stage;
}
```

### Ограничения без TestFX:
- Нет `clickOn()`, `write()`, `lookup()` методов
- Нужно использовать JavaFX API напрямую (Node.lookup(), Robot, InputEvent)
- Больше boilerplate кода для взаимодействия с UI

**Рекомендация:** Рассмотреть использование [TestFXFX](https://github.com/testfx/testfxfx) - альтернативный fork TestFX, если он будет доступен.
