# План реализации: Добавление сохранения настроек таблицы

## Задача
Доработать классы `OrderView` и `DictionaryView` для поддержки сохранения и восстановления настроек таблиц (ширина колонок, порядок, сортировка) согласно инструкции в `docs/table-settings-guide.md`.

## Проблема и Решение для OrderView

### Проблема
Класс `OrderView` имел неправильную инициализацию `masterData`:
- `masterData` инициализировался пустым списком **до** создания таблицы
- `filteredOrders` оборачивал этот пустой список
- При вызове `refreshOrderList()` создавались новые списки, но это нарушало паттерн

### Решение
Следуя паттерну `ClientView`:

#### 1. Удалена инициализация masterData
**Было:**
```java
private static ObservableList<WorkOrder> masterData = FXCollections.observableArrayList();
```

**Стало:**
```java
private static ObservableList<WorkOrder> masterData;
```

#### 2. Исправлено создание FilteredList в create()
**Было:**
```java
filteredOrders = new FilteredList<>(masterData, p -> true);  // masterData пустой!
```

**Стало:**
```java
filteredOrders = new FilteredList<>(FXCollections.observableArrayList(DataStore.getOrders()), p -> true);
```

#### 3. Правильное обновление в refreshOrderList()
```java
public static void refreshOrderList() {
    if (orderTable != null) {
        masterData = FXCollections.observableArrayList(DataStore.getOrders());
        filteredOrders = new FilteredList<>(masterData, p -> true);
        sortedOrders = new SortedList<>(filteredOrders);
        sortedOrders.comparatorProperty().bind(orderTable.comparatorProperty());
        orderTable.setItems(sortedOrders);
        applyFilters();
    }
}
```

## Проблема и Решение для DictionaryView (три таблицы)

### Проблемы
1. **Tab "Услуги" (servicesTable):** 
   - Создавалась локальная переменная `filteredServices` вместо использования поля класса
   - `masterData` объявлен как поле, но использовался внутри метода
   - Колонки не имели `setId()` для сохранения состояния
   - Нет вызова `loadTableState`

2. **Tab "Запчасти" (sparePartsTable):**
   - `filterSpareParts()` не использовал SortedList - просто заменял items
   - Колонки не имели `setId()`
   - Нет FilteredList/SortedList

3. **Tab "Склад" (stockTable):**
   - Просто setItems без FilteredList/SortedList
   - Колонки не имели `setId()`
   - Нет вызова `loadTableState`

### Решение

#### 1. Объявлены поля класса
```java
private static ObservableList<Service> masterData;
private static FilteredList<Service> filteredServices;
private static SortedList<Service> sortedServices;
private static FilteredList<SparePart> filteredSpareParts;
private static SortedList<SparePart> sortedSpareParts;
private static TableView<SparePart> stockTable;
```

#### 2. Исправлены колонки для всех таблиц
Каждой колонке добавлен `setId()`:
```java
colName.setId("colName");
colSparePart.setId("colSparePart");
colDuration.setId("colDuration");
// и т.д.
```

#### 3. Tab "Услуги": createServicesPanel()
```java
// Создаём FilteredList для фильтрации данных
filteredServices = new FilteredList<>(FXCollections.observableArrayList(DataStore.getServices()), p -> true);

// Создаём SortedList для поддержки сортировки
sortedServices = new SortedList<>(filteredServices);
sortedServices.comparatorProperty().bind(table.comparatorProperty());
table.setItems(sortedServices);

servicesTable = table;  // Сохраняем ссылку на таблицу
// Загрузка состояния таблицы
Platform.runLater(() -> {
    if (servicesTable != null) {
        TableStateManager.loadTableState(servicesTable, "servicesTable");
    }
});
```

#### 4. Tab "Запчасти": createSparePartsPanel()

#### 6. Добавлены методы refresh
```java
public static void refreshServicesList() {
    if (servicesTable != null) {
        masterData = FXCollections.observableArrayList(DataStore.getServices());
        filteredServices = new FilteredList<>(masterData, p -> true);
        sortedServices = new SortedList<>(filteredServices);
        sortedServices.comparatorProperty().bind(servicesTable.comparatorProperty());
        servicesTable.setItems(sortedServices);
    }
}

public static void refreshSparePartsList() {
    if (sparePartsTable != null) {
        ObservableList<SparePart> sparePartData = FXCollections.observableArrayList(DataStore.getSpareParts());
        filteredSpareParts = new FilteredList<>(sparePartData, p -> true);
        sortedSpareParts = new SortedList<>(filteredSpareParts);
        sortedSpareParts.comparatorProperty().bind(sparePartsTable.comparatorProperty());
        sparePartsTable.setItems(sortedSpareParts);
        // Применяем текущий фильтр поиска
        if (searchField != null && searchField.getText() != null && !searchField.getText().isEmpty()) {
            filterSpareParts(searchField.getText());
        }
    }
}

public static void refreshStockList() {
    if (stockTable != null) {
        ObservableList<SparePart> stockData = FXCollections.observableArrayList(DataStore.getSpareParts());
        FilteredList<SparePart> filteredStock = new FilteredList<>(stockData, p -> true);
        SortedList<SparePart> sortedStock = new SortedList<>(filteredStock);
        sortedStock.comparatorProperty().bind(stockTable.comparatorProperty());
        stockTable.setItems(sortedStock);
    }
}
```

## Структура данных (трехуровневая архитектура)
```
DataStore.getXxx()
    ↓
ObservableList<Xxx> masterData (источник)
    ↓
FilteredList<Xxx> filteredXxx (фильтрация)
    ↓
SortedList<Xxx> sortedXxx (сортировка + привязка к TableView)
    ↓
TableView<Xxx> table (отображение)
```

## Интеграция с TableStateManager
- `getXxxTable()` — getter для получения таблицы извне
- `Platform.runLater()` — загрузка состояния после отрисовки
- `refreshXxxList()` — пересоздание всех списков при обновлении
- `UNCONSTRAINED_RESIZE_POLICY` — позволяет сохранять ширину колонок

## Ключевые моменты
1. **Сортировка** работает только с `SortedList` + `comparatorProperty().bind()`
2. **Ширина колонок** сохраняется через `getWidth()` и восстанавливается через `setMinWidth()`/`setMaxWidth()` с последующим снятием ограничений
3. **Refresh** должен пересоздавать все три списка (masterData, filteredXxx, sortedXxx), иначе сортировка сломается
4. **Колонки** должны иметь уникальные `id()` для корректной работы TableStateManager
5. **Фильтрация** через `setPredicate()` на FilteredList, а не замена items
