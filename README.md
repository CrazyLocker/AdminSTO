# AdminSTO - Система управления автосервисом

## Описание

AdminSTO - это современное приложение для управления автосервисом, разработанное на Java с использованием JavaFX для графического интерфейса и SQLite для хранения данных.

### Основные функции

- **Управление заказами-нарядами** - создание, редактирование, поиск и печать заказов
- **Управление клиентами** - база данных клиентов с историей обращений
- **Справочники** - услуги и запчасти с возможностью импорта из CSV, XML, JSON
- **Календарь записей** - планирование ремонтных работ
- **Статистика и отчёты** - ежедневная выручка, отчёты по выполненным работам
- **Управление складом** - учёт запчастей, контроль остатков, приход

### Технологический стек

- **Java 17+**
- **JavaFX 17+** - для создания современного GUI
- **SQLite** - легковесная база данных
- **HikariCP** - пул соединений
- **Maven** - управление зависимостями и сборкой

### Архитектура

Проект использует многослойную архитектуру:

- **Model** - модели данных (Order, Client, Part, Service)
- **Controller** - бизнес-логика и управление сценами
- **View** - JavaFX сцены и контроллеры интерфейса
- **Service** - сервисы для работы с данными (StatisticsService, ImportService)
- **Database** - слой доступа к базе данных (H2Database, SQLiteDatabase)

## Установка и запуск

### Требования

- Java Development Kit (JDK) 17 или выше
- Maven 3.6 или выше

### Сборка проекта

```bash
mvn clean package
```

### Запуск приложения

```bash
mvn javafx:run
```

### Исполняемый JAR

После сборки будет создан исполняемый JAR файл в папке `target/`:
- `autoservice.jar` - основной JAR файл
- `autoservice-jar-with-dependencies.jar` - JAR со всеми зависимостями

## Тестирование

В проекте используются модульные тесты с JUnit 5 и AssertJ:

```bash
mvn test
```

### Покрытие кода тестами

Используется JaCoCo для анализа покрытия кода тестами:

```bash
mvn clean test jacoco:report
```

Отчёт о покрытии будет сгенерирован в `target/site/jacoco/index.html`.

## Форматы данных

### Даты

В приложении используется формат даты `dd/MM/yyyy` для отображения и сохранения в базе данных.

### CSV импорт

Формат CSV для импорта запчастей:
```
имя;артикул;производитель;цена;закупка;остаток;место;модели
```

### XML импорт

```xml
<spareParts>
  <part>
    <name>Название запчасти</name>
    <partNumber>Артикул</partNumber>
    <manufacturer>Производитель</manufacturer>
    <retailPrice>Цена</retailPrice>
    <purchasePrice>Закупка</purchasePrice>
    <stock>Остаток</stock>
    <location>Место хранения</location>
    <compatibleModels>Модели</compatibleModels>
  </part>
</spareParts>
```

### JSON импорт

```json
{
  "spareParts": [
    {
      "name": "Название запчасти",
      "partNumber": "Артикул",
      "manufacturer": "Производитель",
      "retailPrice": 1000,
      "purchasePrice": 700,
      "stock": 10,
      "location": "Место",
      "compatibleModels": "Модели"
    }
  ]
}
```
