# План рефакторинга: Удаление справочников и перенос вкладок в главное окно

## Текущее состояние

### Структура Dictionary
- `DictionaryView.java` - содержит 3 вкладки:
  - **Услуги** (createServicesPanel()) - управление услугами
  - **Запчасти** (createSparePartsPanel()) - управление запчастями
  - **Склад** (createStockPanel()) - управление складом

- `DictionaryController.java` - контроллер для всех трех вкладок
- Таблицы инициализируются статическими методами в DictionaryView

### Текущее использование в App.java
В главном окне текущие вкладки уже создают панели из DictionaryView:
```java
servicesTab.setContent(DictionaryView.createServicesPanel());
sparePartsTab.setContent(DictionaryView.createSparePartsPanel());
stockTab.setContent(DictionaryView.createStockPanel());
```

## Требуемое изменение

### Архитектура (утверждено пользователем)
- **Полностью автономные панели** с собственными контроллерами
- **DictionaryView и DictionaryController полностью удаляются**
- **Сохраняем статические геттеры для TableStateManager**

### Новые классы (6 штук)

#### View-классы (3)
1. **ServicePanel.java** - панель управления услугами
2. **SparePartPanel.java** - панель управления запчастями  
3. **StockPanel.java** - панель управления складом

#### Controller-классы (3)
1. **ServicePanelController.java** - логика для услуг
2. **SparePartPanelController.java** - логика для запчастей
3. **StockPanelController.java** - логика для склада

### Панель-контроллер связи
Каждый Panel содержит Table и свой Controller:
- ServicePanel → ServicePanelController
- SparePartPanel → SparePartPanelController  
- StockPanel → StockPanelController

Контроллеры обновляют DataStore и вызывают refresh у своих таблиц.
