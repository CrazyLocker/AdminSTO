## 1. Общее описание

### 1.1. Назначение системы

AdminSTO — это настольное приложение для автоматизации рабочих процессов автосервиса. Система предназначена для управления клиентами, заказами на обслуживание, запасными частями, услугами и записями на сервисное обслуживание.

### 1.2. Область применения

Система применяется в небольших и средних автосервисах для:
- Ведения клиентской базы
- Оформления заказов-нарядов
- Учёта услуг и запасных частей
- Планирования записей на обслуживание
- Генерации отчётности
- Резервного копирования с настраиваемыми параметрами
- Сохранения состояний таблиц (шаги, порядок, сортировка)

### 1.3. Целевая аудитория

- Администраторы автосервиса
- Мастера-приёмщики
- Руководители СТО

### 1.4. Определения и сокращения

| Термин | Определение |
|--------|-------------|
| Заказ-наряд | Документ, фиксирующий работы и запчасти по обслуживанию автомобиля |
| Запчасть | Деталь или расходный материал для ремонта автомобиля |
| Услуга | Работа, выполняемая специалистами автосервиса |
| Запись | Запланированное посещение автосервиса клиентом |
| DataStore | Единый центр управления данными приложения |
| CRUD | Create, Read, Update, Delete — основные операции с данными |

### 2.2. Компоненты системы

#### 2.2.1. Presentation Layer (Представление)

| Компонент | Описание |
|-----------|----------|
| `DashboardView` | Главная панель с метриками и статистикой |
| `ClientView` | Интерфейс управления клиентами |
| `OrderView` | Интерфейс управления заказами-нарядами |
| `DictionaryView` | Управление справочниками услуг и запчастей |
| `AppointmentView` | Календарь записей на обслуживание |
| `SettingsView` | Настройки системы |
| `ReportView` | Генерация и просмотр отчётов |
| `Dialogs` | Модальные окна для CRUD-операций |

#### 2.2.2. Controller Layer (Контроллеры)

| Компонент | Описание |
|-----------|----------|
| `ClientController` | Обработка действий пользователя в разделе клиентов |
| `OrderController` | Обработка действий пользователя в разделе заказов |
| `DictionaryController` | Управление справочниками |
| `SettingsController` | Управление настройками системы |

#### 2.2.3. Service Layer (Бизнес-логика)

| Компонент | Описание |
|-----------|----------|
| `ImportService` | Импорт данных из внешних источников (CSV) |
| `StatisticsService` | Расчёт статистики для дашборда |
| `SettingService` | Управление настройками приложения (база данных) |
| `BackupService` | Резервное копирование БД и состояний таблиц |
| `AutoAddSparePartService` | Автоматическое добавление запчастей |
| `ScheduleService` | Планирование автоматических бэкапов

#### 2.2.4. Data Layer (Данные)

| Компонент | Описание |
|-----------|----------|
| `DataStore` | Единая точка доступа к данным, кэширование |
| `Database` | Фабрика и менеджер подключений к БД |
| `SQLiteDatabase` | Реализация работы с SQLite |
| `H2Database` | Реализация для тестирования (in-memory) |
| `DatabaseInterface` | Интерфейс доступа к данным |
| `AbstractDatabase` | Базовая реализация общих методов |
| `TableStateManager` | Сохранение и восстановление состояний таблиц (шаги, ширины, сортировка)

#### 2.2.5. Model Layer (Модели)

| Модель | Описание |
|--------|----------|
| `Client` | Клиент автосервиса |
| `Service` | Услуга автосервиса |
| `SparePart` | Запчасть/расходный материал |
| `WorkOrder` | Заказ-наряд |
| `Appointment` | Запись на обслуживание |
| `Setting` | Настройка системы |
| `ServiceSparePart` | Связь услуг и запчастей |

#### 2.2.6. Utility Layer (Утилиты)

| Утилита | Описание |
|---------|----------|
| `DateUtils` | Работа с датами (форматирование, парсинг) |
| `Validators` | Валидация данных (телефоны, госномера) |
| `OilHelper` | Вспомогательные методы для работы с маслом |
| `IconHelper` | Управление SVG-иконками |
| `ReportGenerator` | Генерация PDF-отчётов |
| `ExceptionHandler` | Глобальный обработчик исключений для UI |
| `ValidationUtils` | Валидация форм и полей |
| `ValidationErrorIndicator` | Визуальная индикация ошибок валидации

## 5. Модель данных

### 5.1. Схема базы данных

```sql
-- Таблица клиентов
CREATE TABLE clients (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    phone TEXT NOT NULL,
    car_model TEXT,
    car_number TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Таблица услуг
CREATE TABLE services (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    price REAL NOT NULL CHECK(price >= 0),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Таблица запчастей
CREATE TABLE spare_parts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    purchase_price REAL NOT NULL CHECK(purchase_price >= 0),
    retail_price REAL NOT NULL CHECK(retail_price >= 0),
    stock INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Таблица заказов
CREATE TABLE work_orders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    client_id INTEGER NOT NULL,
    status TEXT NOT NULL,
    total_amount REAL NOT NULL DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (client_id) REFERENCES clients(id) ON DELETE CASCADE
);

-- Связь заказов и услуг
CREATE TABLE order_services (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    order_id INTEGER NOT NULL,
    service_id INTEGER NOT NULL,
    quantity INTEGER NOT NULL DEFAULT 1,
    price REAL NOT NULL,
    FOREIGN KEY (order_id) REFERENCES work_orders(id) ON DELETE CASCADE,
    FOREIGN KEY (service_id) REFERENCES services(id)
);

-- Связь заказов и запчастей
CREATE TABLE order_spare_parts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    order_id INTEGER NOT NULL,
    spare_part_id INTEGER NOT NULL,
    quantity INTEGER NOT NULL DEFAULT 1,
    price REAL NOT NULL,
    FOREIGN KEY (order_id) REFERENCES work_orders(id) ON DELETE CASCADE,
    FOREIGN KEY (spare_part_id) REFERENCES spare_parts(id)
);

-- Таблица записей
CREATE TABLE appointments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    client_id INTEGER NOT NULL,
    master_name TEXT,
    service_name TEXT,
    appointment_date TEXT NOT NULL,
    appointment_time TEXT NOT NULL,
    status TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (client_id) REFERENCES clients(id) ON DELETE CASCADE
);

-- Таблица настроек
CREATE TABLE settings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    key TEXT UNIQUE NOT NULL,
    value TEXT,
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Таблица расходников заказа
CREATE TABLE order_to_parts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    order_id INTEGER NOT NULL,
    spare_part_id INTEGER NOT NULL,
    quantity INTEGER NOT NULL DEFAULT 1,
    price REAL NOT NULL,
    car_model TEXT,
    FOREIGN KEY (order_id) REFERENCES work_orders(id) ON DELETE CASCADE,
    FOREIGN KEY (spare_part_id) REFERENCES spare_parts(id)
);
```

## Новые возможности версии 1.0.0

### 1. Редактируемая модель автомобиля в TO Parts

При редактировании расходников в заказе-наряде модель автомобиля теперь поддерживает редактирование:
- **Предопределённый список**: 13 популярных моделей GWM (Haval Jolion, F7, F7x, Dargo, Big Dog, H6, H9, Poer, Tank 300, Tank 500, Wingle 7, Cannon, Great Wall Poer)
- **Ручной ввод**: Возможность ввести кастомную модель автомобиля, если нужная отсутствует в списке
- **Поведение**: Комбобокс становится редактируемым (`setEditable(true)`), значение берётся через `getEditor().getText()`

### 2. Настройки резервного копирования

Добавлены настраиваемые параметры резервного копирования:

| Параметр | Описание |
|----------|----------|
| `autoBackupEnabled` | Включение/отключение автоматического резервного копирования |
| `backupTime` | Время создания ежедневных бэкапов (HH:mm формат) |
| `backupRetention` | Количество хранимых бэкапов (по умолчанию 14) |

**Функции `BackupService`:**
- `createBackup()` — создаёт ZIP-архив с БД и файлами `config/table-state/*.json`
- `cleanupOldBackups()` — автоматически удаляет старые бэкапы
- `restoreBackup(path)` — восстанавливает из резервной копии

**Структура бэкапа:**
```
backup_2026-07-15_14-30-00.zip
├── autoservice.db
├── table-state/
│   ├── clientTable.json
│   ├── orderTable.json
│   └── ...
└── ...
```

### 3. Сохранение состояний таблиц

Каждая таблица в системе сохраняет своё состояние:
- **Ширина колонок** — пользовательская ширина колонок
- **Порядок колонок** — порядок отображения колонок
- **Видимость колонок** — какие колонки показаны/скрыты
- **Сортировка** — текущая сортировка (по какой колонке, ASC/DESC)

**Файлы состояний:**
```
config/table-state/
├── clientTable.json
├── orderTable.json
├── serviceSparePartsTable.json
├── servicesTable.json
├── settingsTable.json
├── sparePartsTable.json
├── stockTable.json
└── toPartsTable.json
```

**Архитектура:**
- `TableStateManager` — управление состояниями
- `TableState` / `ColumnState` / `SortState` — структуры данных
- `Gson` — JSON сериализация
