# Инструкция для разработчика: Анализ и доработка системы связей "Услуга → Запчасти"

## 1. Цель задачи
Перейти от жёстко зашитых в коде связей (только для масла и фильтров) к **гибкой системе**, где связи между услугами и запчастями настраиваются через интерфейс и хранятся в базе данных.

Это позволит:
- Добавлять любые запчасти к любым услугам
- Указывать разное количество (литры, штуки)
- Делать запчасти обязательными или опциональными
- Менять связи без перекомпиляции

---

## 2. Текущее состояние (что уже есть)

### 2.1. Модели данных
- **`Service.java`** — содержит поля: `oilVolume`, `usesOil`, `sparePartName`, `sparePartQuantity`.
- **`SparePart.java`** — содержит поля: `stock`, `unitType`, `unitVolume`, `isLiquid`.
- **`WorkOrder.java`** — хранит услуги и запчасти в заказе.

### 2.2. Сборка и логика
- **`OilHelper.java`** — содержит методы для автоматического добавления масла и фильтров при создании заказа (жёсткая логика).
- **`CreateOrderDialog.java`** и **`EditOrderDialog.java`** — используют `OilHelper` для добавления запчастей.

### 2.3. База данных
- **Таблица `services`** — содержит поля `oil_volume`, `uses_oil`, `spare_part_name`, `spare_part_quantity`.
- **Таблица `spare_parts`** — содержит поля `stock`, `unit_type`, `unit_volume`, `is_liquid`.
- **Таблица `order_parts`** — хранит запчасти в заказе (без связи с `service_parts`).

---

## 3. Недостатки текущей реализации

| Проблема | Описание |
|----------|----------|
| **Только для масла** | Система работает только с маслом и фильтрами. |
| **Только одна запчасть** | Для услуги можно указать только одну запчасть (`spare_part_name`). |
| **Нет опциональности** | Все запчасти обязательны, нельзя сделать опциональными. |
| **Изменения в коде** | Любое изменение требует перекомпиляции и пересборки. |
| **Нет интерфейса** | Нельзя добавлять/редактировать связи через UI. |
| **Нет истории** | Не отслеживается, когда и кто добавил связь. |

## 6. Данные для проверки (сценарии)

### 6.1. Проверить существующие данные
- Убедиться, что все услуги и запчасти загружаются корректно
- Проверить, что старые связи (масло, фильтры) перенесены в новую таблицу
- Убедиться, что удаление устаревших полей не ломает существующие запросы

### 6.2. Проверить добавление связи
- Открыть вкладку "Связи"
- Добавить связь: Услуга "Замена моторного масла" → Запчасть "Масло 5W-30" → Количество 5.5 → Обязательно
- Добавить ещё одну связь: Услуга "Замена моторного масла" → Запчасть "Фильтр масляный" → Количество 1 → Обязательно
- Убедиться, что связи сохранились и отображаются в таблице

### 6.3. Проверить создание заказа
- Создать новый заказ
- Добавить услугу "Замена моторного масла"
- Убедиться, что в заказ автоматически добавились:
    - Масло 5W-30 (5.5 л)
    - Фильтр масляный (1 шт)
- Проверить, что остатки на складе уменьшились корректно

### 6.4. Проверить опциональную запчасть
- Добавить связь: Услуга "Замена моторного масла" → Запчасть "Шайба сливная" → Количество 1 → Не обязательно
- Создать заказ с этой услугой
- Убедиться, что шайба добавляется, но не блокирует создание заказа при отсутствии на складе

### 6.5. Проверить разные услуги
- Создать заказ с услугами:
    - "Замена моторного масла" (должны добавиться масло, фильтр)
    - "Замена масляного фильтра" (должен добавиться фильтр, но только один, без дублей)
    - "Замена передних колодок" (если есть связь — добавится)
- Убедиться, что дублирующиеся запчасти объединяются

### 6.6. Проверить редактирование заказа
- Открыть существующий заказ
- Удалить услугу, добавить другую
- Убедиться, что запчасти пересчитываются корректно

## 7. Что должно НЕ сломаться

| Сценарий | Ожидание |
|----------|----------|
| Создание заказа без услуг | ✅ Работает, показывается предупреждение |
| Создание заказа с услугами, но без запчастей | ✅ Добавляются только услуги, запчасти не обязательны |
| Редактирование заказа с уже добавленными запчастями | ✅ Запчасти обновляются корректно |
| Удаление заказа | ✅ Запчасти возвращаются на склад |
| Закрытый заказ | ✅ Нельзя редактировать |
| Отсутствие запчастей на складе | ✅ Предупреждение, заказ создаётся (для опциональных) или блокируется (для обязательных) |

---

## 8. Критерии успеха

| Критерий | Проверка |
|----------|----------|
| **Гибкость** | Можно добавить связь для любой услуги и запчасти через интерфейс |
| **Надёжность** | При создании заказа все обязательные запчасти добавляются автоматически |
| **Скорость** | Запросы к базе данных оптимизированы (индексы, JOIN) |
| **Чистота кода** | Удалён устаревший код, соблюдён принцип SRP |
| **Обратная совместимость** | Существующие заказы не ломаются |
