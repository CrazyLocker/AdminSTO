# Инструкция по использованию portable-версии

## Что создано

Папка `dist/` содержит полностью автономную portable-версию приложения:

```
dist/
├── run.bat              ← Запуск приложения
├── autoservice-admin.jar (35 МБ) — Приложение со всеми зависимостями
├── README.txt           — Краткая справка
├── javafx/              — JavaFX для Windows
│   ├── javafx-base-17.0.6-win.jar
│   ├── javafx-controls-17.0.6-win.jar
│   ├── javafx-fxml-17.0.6-win.jar
│   └── javafx-graphics-17.0.6-win.jar
└── data/                — Папка для данных (опционально)
```

## Как использовать

### 1. Копирование на флешку
```
Скопируйте всю папку dist целиком на флешку
```

### 2. Запуск на другом компьютере
```
1. Вставьте флешку
2. Перейдите в папку dist
3. Запустите run.bat (двойной клик)
```

### 3. Требования к целевому компьютеру
```
- Windows 10/11 (64-bit)
- JDK 17 или выше (ОБЯЗАТЕЛЕН)
- ОЗУ: 512 МБ минимум
- Место на диске: ~150 МБ
```
PS> Get-ChildItem "dist\jdk" -Recurse | Measure-Object -Property Length -Sum | Select-Object @{N='SizeMB';E={[math]::Round($_.Sum/1MB,2)}}
## Если run.bat не работает

### Проблема: "JDK 17+ not found"
**Решение:**
1. Установите JDK 17+ с https://adoptium.net/
2. Или настройте переменную среды JAVA_HOME

### Проблема: Приложение не запускается
**Решение:**
1. Запустите run.bat из командной строки (cmd)
2. Посмотрите сообщения об ошибках
3. Проверьте права на запись в папку

## Где хранятся данные

```
База данных: autoservice.db (создается при первом запуске)
Расположение: В папке с приложением (dist/)
```

### Резервное копирование
```
Для бэкапа просто скопируйте файл autoservice.db
```

## Как пересобрать portable-версию

```powershell
# В папке проекта выполните:
powershell -ExecutionPolicy Bypass -File build-portable.ps1
```

Или вручную:
```
1. mvn clean package -DskipTests
2. Скопируйте JAR в dist/
3. Скопируйте JavaFX binaries из Maven репозитория
4. Запустите build-portable.ps1
```

## Структура проекта

```
AdminSTO/
├── dist/                  ← Portable-версия (для флешки)
├── src/                   ← Исходный код
├── target/                ← Собранные файлы
├── build-portable.ps1     ← Скрипт сборки portable
├── build-portable.bat     ← Альтернативный скрипт (bat)
└── pom.xml                ← Maven конфигурация
```

## Примечания

1. **JavaFX требуется**: Приложение использует JavaFX, поэтому нужен `--module-path`
2. **База данных SQLite**: Хранится в файле, легко бэкапится
3. **Относительные пути**: Все пути к данным относительные, работает с любой папки
4. **Зависимости**: Все зависимости встроены в fat JAR

## Поддержка

При проблемах:
1. Убедитесь, что JDK установлен
2. Запустите run.bat из cmd для просмотра ошибок
3. Проверьте права на запись
